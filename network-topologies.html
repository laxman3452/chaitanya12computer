CHAPTER 2: DATA COMMUNICATION & NETWORKING

Complete Exam Notes: Network Topologies

1. Introduction to Network Topologies

What is Network Topology?

Definition: Network topology is the physical or logical layout representing the geometrical pattern or arrangement of how computers (nodes) are interconnected in a network.

Simply put, it describes how computers are connected and how data flows between them.

Types of Network Topologies

While there are several topologies (Bus, Star, Ring, Mesh, Tree, and Hybrid), school curriculums focus primarily on the three core types:

Bus Topology Star Topology Ring Topology

2. Bus Topology (Linear Topology)

Core Concept

In a Bus topology, all computers and network devices are connected to a single central cable called the trunk or bus.

  • Transmission Media: Coaxial cables are primarily used.
  • Terminators: Placed at both ends of the main cable. They absorb signals to prevent them from bouncing back (reflecting) and causing data interference.
  • Data Flow: It uses a broadcasting mechanism where data sent by one computer is transmitted to all other computers on the network, though only the intended receiver processes it.

Bus Topology Diagram

Advantages

  • Easy to Implement & Extend: Simple to set up and add new devices.
  • Cheapest Option: Requires a minimal amount of cabling, making it highly cost-effective.
  • Great for Temporary Networks: Very easy to set up quickly for temporary use.
  • Node Independence: If a single computer (node) fails, the rest of the network continues to work normally.

Disadvantages

  • Single Point of Failure: If the main cable (trunk) breaks, the entire network goes down.
  • Difficult Troubleshooting: Pinpoint-finding a fault or cable break can be extremely difficult.
  • Traffic & Collisions: High data traffic increases the chance of data collisions and error rates.
  • Limited Scale: Limit on cable length and the total number of nodes that can be connected.

3. Star Topology (The Most Popular Topology)

Core Concept

Each computer (node) is connected directly to a central device called a Hub or Switch using a dedicated cable.

  • Transmission Media: Twisted-pair cables are most commonly used.
  • Data Flow: Data sent from a computer must pass through the central hub/switch first, which then directs or broadcasts it to the destination.

Star Topology Diagram

Advantages

  • Easy Management: Very simple to add or remove computers without disrupting the rest of the network.
  • Highly Reliable: If one cable or node fails, only that device is disconnected. The rest of the network is completely unaffected.
  • Easy Troubleshooting: Faults are incredibly easy to locate and fix because every computer has its own line.
  • Centralized Control: Allows administrators to monitor and manage the network from the central hub.

Disadvantages

  • Central Failure: If the central hub or switch fails, the entire network goes down.
  • High Cost: More expensive than bus topology because of the cost of the hub/switch and a larger amount of cabling.
  • Heavy Cable Requirement: Every single computer requires its own dedicated cable, which can lead to a messy web of wires if not properly organized.

4. Ring Topology

Core Concept

All computers are connected to nearby computers on a point-to-point basis, forming a closed circular loop.

  • Data Flow: Data travels in one direction only (either clockwise or counter-clockwise).
  • Token Passing: It uses a unique method called "token passing" to transmit data. Only the computer holding the digital "token" is allowed to send data, preventing collisions.
  • Central Device: A MAU (Multi-station Access Unit) is often used as a control device.
  • Transmission Media: Twisted pair or optical fiber cables are used.

Ring Topology Diagram

Advantages

  • No Collisions: Because data flows in one direction and uses token passing, there are no data collisions.
  • Equal Priority: Every computer has an equal opportunity to transmit data.
  • No Terminators Needed: The circular loop design eliminates the need for terminators.
  • Performance Stability: Even as more devices are added, the impact on network speed is minimal.

Disadvantages

  • Whole-Network Failure: If a single computer or cable link breaks, the entire network is disabled because the loop is broken.
  • Difficult Installation: Harder to configure and set up compared to Bus or Star.
  • Expensive: The requirement of specialized hardware like MAUs makes it costlier than Bus.
  • Inflexible: Adding or removing devices disrupts the network because you have to break the ring.

6. Selection Criteria for Network Topology

When designing a network, engineers choose a topology based on these 8 key factors:

1. CostBudget for cables, central devices, and installation.
2. Data Error RateHow prone the layout is to interference and packet loss.
3. SimplicityHow easy it is to install, configure, and understand.
4. Data SecurityHow easy it is to intercept data traveling through the layout.
5. Transmission SpeedThe rate at which data travels.
6. FlexibilityHow easily the network can expand or change.
7. MaintainabilityEase of managing and repairing the system.
8. Volume of Data TrafficThe amount of data flowing through the network at any given time.

7. Why is Star Topology Preferred for LANs in Organizations?

In exams, this is a highly common long-answer question. Here are the 4 pillars of why Star topology is preferred:

High Reliability

In an office, if one employee's computer cable gets damaged, only their computer loses connection. The rest of the office can work without interruption.

High Performance

Using a central switch allows dedicated lanes of communication. This virtually eliminates data collisions, ensuring high speeds even during peak office hours.

Cost-Effective Scalability

Unlike complex mesh topologies, Star only requires one port and one cable per device. It scales easily as the organization grows.

Easy Troubleshooting & Maintenance

IT administrators can easily spot which port on the switch is showing an error, allowing them to pinpoint and resolve cable or node faults instantly.
