1.1 Data, Information, Database & DBMS

Data: Data is a collection of raw facts, figures, or symbols that do not have a specific meaning on their own.
Example: 1, Rubina, 5.4
Information: Processed, organized, and structured data that carries a clear and complete meaning.
Example: "The height of Rubina is 5.4 feet."

Difference between Data and Information (Highly Important for Exam)

Features Data Information
Meaning Raw facts without meaning. Processed data with a clear meaning.
Input/Output It serves as the input to a system. It is the final output of processing.
Dependency Does not depend on information. Completely depends on data.
Database: A systematic collection of interrelated data stored logically in tabular form (rows and columns).
Example: Phone book, student mark sheet, library register.
DBMS (Database Management System): A software system used to create, manage, manipulate, and retrieve data from a database safely.
Popular Examples: MySQL, Oracle, MS-Access, MS-SQL Server, PostgreSQL.

Definitions to write in exam:

• Primary Key: A specific single field (or combination of fields) that uniquely identifies each row in a table. It **cannot be NULL** and **cannot contain duplicate values**.

• Candidate Key: All columns or combinations of columns that have the potential capability to become a primary key (uniquely identify rows).

• Alternate Key: The remaining candidate keys that were not chosen to be the primary key. (Also known as Secondary Keys).

1.3 Advantages and Disadvantages of DBMS

Advantages (Write these 4 points for full marks)

  • Reduced Data Redundancy: It minimizes data duplication, saving storage room and avoiding conflicts.
  • Easy Data Sharing: Multiple authorized users can access the exact same dataset simultaneously.
  • Data Integrity: Enforces rules and constraints so that data remains accurate and consistent.
  • Data Security: Protects private records using proper login access controls and permissions.
  • Data Backup & Recovery: Provides automated tools to restore records safely if hardware fails.

Disadvantages

  • High Cost: Requires expensive software licensing, setup configurations, and hardware.
  • Technical Complexity: Requires highly skilled and trained professionals (DBAs) to handle it.
  • High Impact of Failure: Since data is fully centralized, if the database crashes, all related applications stop working instantly.
