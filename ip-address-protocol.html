1. Internet Protocol (IP)

Definition: Internet Protocol (IP) is a communication protocol used to identify devices and deliver data from a source device to a destination device over a network.
Exam Setup Definition Internet Protocol (IP) is a set of rules used to identify devices and transfer data over a network.

2. IP Address

Definition: An IP Address is a unique logical numerical address assigned to every device connected to a network so that devices can communicate with each other.

Examples

  • 192.168.1.10
  • 172.16.5.20
  • 10.0.0.5

Functions of an IP Address

  • Identifies every device uniquely.
  • Enables communication between devices.
  • Helps routers deliver data to the correct destination.
  • Identifies both the network and the host.

3. Why is an IP Address Needed?

Without an IP address framework:

  • Devices cannot identify each other.
  • Data cannot reach the correct destination.
  • Internet communication is impossible.
Real-World Analogy: A physical house needs a structured mailing address to receive letters correctly. Similarly, a computer or node needs an IP address to receive digital data packets over the web.

4. IPv4 (Internet Protocol Version 4)

IPv4 is the most widely deployed and commonly recognized version of the Internet Protocol.

Characteristics

  • 32-bit total address length.
  • Divided into 4 equal octets.
  • Each individual octet contains exactly 8 bits.
  • Uses decimal numbers separated by dots (.).

Structural Blueprint

192 . 168 . 1 . 25
8 bits | 8 bits | 8 bits | 8 bits ==> Total = 32 bits

Note: The decimal value of each individual octet can strictly range from 0 to 255.

5. IPv6 (Internet Protocol Version 6)

IPv6 is the newer generation version of IP developed primarily to replace IPv4 due to the imminent global shortage of available IPv4 address spaces.

Characteristics

  • 128-bit long address block.
  • Uses hexadecimal numbers instead of decimal.
  • Separated internally by colons (:) instead of dots.

Example

2001:0db8:85a3:0000:0000:8a2e:0370:7334 (Often compressed as 2001:db8:85a3::8a2e:370:7334)

7. Types of IP Addresses

Public IP Address

  • Assigned directly by an Internet Service Provider (ISP).
  • Globally unique and accessible through the open Internet.
  • Example: 103.56.100.12

Private IP Address

Used inside isolated local environments like homes, schools, and offices. They cannot route onto the public web directly.

Reserved Private IP Ranges:

  • 10.0.0.0 – 10.255.255.255
  • 172.16.0.0 – 172.31.255.255
  • 192.168.0.0 – 192.168.255.255

Static IP Address

  • Fixed manually; does not change when the machine restarts.
  • Ideal for hosting core structural servers, corporate printers, network web pages, and CCTV streams.

Dynamic IP Address

  • Leased dynamically and automatically via a DHCP Server.
  • Subject to structural change automatically over time.
  • Standard default profile for consumer laptops, personal smart phones, and typical home users.

8. Network ID and Host ID

Every logical IPv4 address is split into two structural segments:

Network ID  |  Host ID

Example breakdown: 192.168.1.25

  • Network ID: 192.168.1 (Identifies the target subnet context)
  • Host ID: 25 (Identifies the exact physical node target on that subnet)

9. Classful IP Addressing

Historically, legacy IPv4 environments divided structural blocks into five specific architectural tiers determined strictly by evaluating the address's **first octet** boundary.

Class A

  • First Octet Range: 1 – 126
  • Default Subnet Mask: 255.0.0.0
  • Bit Layout: 8 Network bits, 24 Host bits
  • Host Math Allocation: 224 = 16,777,216 total addresses
  • Usable Hosts: 16,777,216 − 2 = 16,777,214 usable devices
  • Primary Intent: Designed for massive multinational organizations and tier-1 ISPs.

Class B

  • First Octet Range: 128 – 191
  • Default Subnet Mask: 255.255.0.0
  • Bit Layout: 16 Network bits, 16 Host bits
  • Host Math Allocation: 216 = 65,536 total addresses
  • Usable Hosts: 65,536 − 2 = 65,534 usable devices
  • Primary Intent: Geared towards medium to large scale enterprises.

Class C

  • First Octet Range: 192 – 223
  • Default Subnet Mask: 255.255.255.0
  • Bit Layout: 24 Network bits, 8 Host bits
  • Host Math Allocation: 28 = 256 total addresses
  • Usable Hosts: 256 − 2 = 254 usable devices
  • Primary Intent: Standard format for default home setups, small offices, and schools.

Class D

  • First Octet Range: 224 – 239
  • Primary Intent: Dedicated solely to Multicast Communication parameters. No default mask.

Class E

  • First Octet Range: 240 – 255
  • Primary Intent: Reserved completely for global research, national labs, and experimental purposes.

11. Why Do We Subtract Two Addresses?

Every operational IP network context must strictly reserve two addresses from its pool for internal management logic:

  • Network Address: The first address in the block. Identifies the network interface itself (All host bits are 0).
  • Broadcast Address: The absolute final address in the block. Used to drop packets out to all nodes inside that network layer concurrently (All host bits are 1).

Structural Breakdown Loop (192.168.1.0/24)

  • 192.168.1.0 → Network Address (Unusable for assignments)
  • 192.168.1.1 down to 192.168.1.254 → Usable Node Hosts
  • 192.168.1.255 → Broadcast Address (Unusable for assignments)

Mathematical Proof: 256 Total Combinations − 2 Fixed Reserves = 254 Real Usable Node Interfaces.

12. Subnet Mask

Definition: A Subnet Mask is a 32-bit number that separates the Network ID and the Host ID of an IP address.
Exam Setup Definition A subnet mask identifies which part of an IP address is the network portion and which part is the host portion.

13. Why is a Subnet Mask Needed?

Consider looking at this unmasked address interface alone: 192.168.1.25

Without an explicit masking value attached, hardware logic cannot dynamically determine exactly where the Network ID boundary ends and the local Host ID interface segment begins.

Applying the Masking Parameter:

  • IP Address: 192.168.1.25
  • Subnet Mask: 255.255.255.0

By mapping these structural items against one another, components read the alignment map as:

192.168.1 | 25
Network Segment       Host Segment

14. Binary Representation of Subnet Mask

How systems execute this evaluation at the logic-gate layer:

Decimal Profile: 255.255.255.0

11111111 . 11111111 . 11111111 . 00000000

Core Binary Structural Evaluation Rules:

  • Binary 1 values indicate network portion bits.
  • Binary 0 values indicate local node host portion bits.

Because there are exactly 24 contiguous binary ones tracked from left to right in this map, the structure is cleanly compressed into shorthand **CIDR notation** as: `/24`

15. Default Subnet Masks

Class Default Subnet Mask CIDR Shorthand Notation
Class A 255.0.0.0 /8
Class B 255.255.0.0 /16
Class C 255.255.255.0 /24

16. Practical Examples

Example 1

IP Address: 192.168.1.25 | Subnet Mask: 255.255.255.0

• Isolated Network ID = 192.168.1.0
• Isolated Host Token = 25

Example 2

IP Address: 172.16.25.100 | Subnet Mask: 255.255.0.0

• Isolated Network ID = 172.16.0.0
• Isolated Host Token = 25.100

Example 3

IP Address: 10.20.30.40 | Subnet Mask: 255.0.0.0

• Isolated Network ID = 10.0.0.0
• Isolated Host Token = 20.30.40
