Data Transmission Impairments & Network Architecture

1. Transmission Impairments Terminology

In data communication, signals travel through transmission media. Because physical media are not perfect, the signals degrade during transit. This imperfection is called transmission impairment, meaning "what is sent is not exactly what is received."

Key Impairment Terms

  • 1. Jitter: Small, irregular time delays during data transfer. It is the variations in arrival time of data packets caused by network congestion, packet collisions, or signal interference.
  • 2. Attenuation: The loss or reduction in the strength/amplitude of a signal as it travels through a cable.
    • It increases with distance.
    • An Amplifier is used to boost signal strength back to original levels to compensate for this loss.
    • It is measured in decibels (dB).
  • 3. Distortion: The change in the original form or shape of a waveform. It usually happens because different frequency components of a composite signal travel at different speeds through the medium.
  • 4. Noise: Random, unwanted, or extra signals that mix with the original transmitted signal and corrupt it. Types include:
    • Thermal noise: Random motion of electrons.
    • Induced noise: From external sources like motors or appliances.
    • Crosstalk: Unwanted signal transfer from an adjacent wire/channel.
    • Impulse noise: Sudden, high-energy spikes (like lightning).
  • 5. Echo: A reflection of a sound or signal that arrives back at the sender/listener with a distinct delay after the direct path signal. It acts as an interfering noise source.
  • 6. Singing: An extreme form of echo that gets completely out of control. It occurs at a circuit's resonant frequency when line losses are so low that the signal oscillates continuously even after the source pulse stops.
  • 7. Crosstalk: Unwanted signals leaking into a communication channel from another nearby circuit. It is caused by unintended capacitive, inductive, or conductive coupling between channels.
  • 8. Bandwidth: The maximum rate of data transfer across a network or internet connection in a given amount of time. For example, Gigabit Ethernet features a bandwidth of 1Gbps (1 Gigabyte per second). Transmission impairments reduce the usable bandwidth.
  • 9. Number of Receivers: Every added node or device connected to a shared transmission line introduces additional attenuation and distortion. Therefore, a higher number of receivers limits the maximum distance and maximum data rate achievable.

2. Basic Concepts of Network Architecture

Network Architecture refers to the physical and logical design of software, hardware, protocols, and transmission media. It explains how computers and network devices interact and how tasks are allocated across the system.

Types of Network Architectures

Computer networks are primarily classified into two main structural architectures:

  1. Peer-to-Peer (P2P) Architecture
  2. Client/Server Architecture

A. Peer-to-Peer (P2P) Architecture

In a peer-to-peer network, all computers have equal authority and rights to access data. There is no dedicated or central server. Every individual computer acts as both a client (requesting services) and a server (providing services). A small group of P2P connected computers is often called a workgroup.

Peer-to-Peer (P2P) Layout

[PC 1: Peer] <=========> [PC 2: Peer] ^ ^ | | v v [PC 4: Peer] <=========> [PC 3: Peer] (All PCs are equal; no central server)

Advantages:

  • Simple design and very easy to set up.
  • Highly cost-effective (cheaper to implement and manage).
  • Fault isolation: If one computer fails, it does not stop the rest of the network.
  • No specialized, dedicated system administrator is required.

Disadvantages:

  • Network administration becomes highly decentralized and difficult.
  • Data security is weak because every user manages their own security settings.
  • Data backup and recovery processes are complex and scattered across nodes.
  • Not suitable for large organizations with a massive number of computers.

B. Client/Server Architecture

This architecture consists of at least one powerful, centralized dedicated server and multiple client computers. The server holds resource databases and provides specialized network services (like data files, web pages, email, SMS, routing, or printing) upon receiving explicit requests from the clients.

Example: A dedicated central file server or a mail server acting as a network "post office" to handle and distribute external communication.

Client/Server Layout

[Client PC] ----\ /---- [Client PC] v v [CENTRAL SERVER] ^ ^ [Client PC] ----/ \---- [Client PC] (Centralized control and services)

Advantages:

  • Centralized Administration: Easier control over all assets from a single point.
  • High Security: Access permissions can be tightly regulated at the server level.
  • Perfectly optimized for large organizations containing numerous endpoints.
  • Simplified Backups: Centralized data means recovery operations are fast and reliable.

Disadvantages:

  • Expensive setup due to the cost of high-end dedicated servers and operating software.
  • Complex configuration, setup, management, and deployment routines.
  • Requires experienced, professional network administrators to operate.
  • Single Point of Failure: If the central server goes down, the entire network stops functioning.

3. Core Differences: P2P vs. Client/Server

S.N. Peer-to-Peer (P2P) Network Architecture Client-Server Network Architecture
1 A simple connection of similar computers called peers. Contains two distinct roles: Centralized Server and Client computers.
