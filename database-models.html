Database Models (Simplified Exam Notes)

Class 12 Computer Science XII — High-Scoring Point-Wise Guide

What is a Database Model? It is a layout design that defines how data is stored, organized, and structurally linked together in a database management system.

1. Hierarchical Database Model

  • Core Meaning: It arranges data records in an upside-down, tree-like structure.
  • The One-Parent Rule: It works on a strict parent-child relationship where a child record can have only one single parent record.
  • The Root Element: The very top-level record is called the parent or root node, and all branches move downwards from it.
Figure 1.1: Hierarchical Tree Structure (Reference: 1000084291.jpg)

💡 Super Easy Example & Usage Area

  • Realistic Example: Computer File System (Your C: Drive → contains a "Pictures" Folder → which contains an "Images" file. The file cannot belong to two folders at the same time).
  • Main Usage Area: Used inside operating system file directories and XML document structural mappings.

✔ Advantages

  • Very Simple: The design is easy to understand and quick to set up.
  • Fast Searches: Data searching is highly efficient if you already know the exact parent track path.

✖ Disadvantages

  • No Many-to-Many Links: It cannot connect a child to multiple parents naturally.
  • High Waste (Redundancy): Identical data must be copied repeatedly across separate branches, wasting memory.

2. Network Database Model

  • Core Meaning: It organizes data records in a circular, interconnected web or graph structure.
  • Multiple Parent Rule: A child record can have multiple parent records, removing the single-parent barrier.
  • The Main Purpose: It was created specifically to fix the rigid relationship limits of the hierarchical model.
Figure 1.2: Network Graph Structure (Reference: 1000084292.jpg)

💡 Super Easy Example & Usage Area

  • Realistic Example: College Course Enrollment (A student is linked to both a Math Teacher and an English Teacher parent node, while both teachers are linked to that same student child node).
  • Main Usage Area: Used in old-school mainframe architectures and complex transaction tracking networks.

✔ Advantages

  • Handles Complex Links: It perfectly maps out many-to-many relationship connections.
  • Less Data Waste: Pointers link records together instead of duplicating files in memory.

✖ Disadvantages

  • Too Complex: Managing hundreds of interconnected data pointers is highly difficult.
  • Hard to Modify: Changing one record requires updating multiple attached system pointers.

3. Relational Database Model

  • Core Meaning: It arranges data elements inside basic grids made of horizontal rows and vertical columns.
  • Key Terms to Remember: Tables are called relations, rows are called tuples, and columns are called attributes.
  • Table Connections: Separate tables link with one another dynamically using matching common field values (like IDs).

💡 Super Easy Example & Usage Area

  • Realistic Example: Digital Spreadsheet Book (An Excel file containing a "Students Info" sheet and a separate "Exam Scores" sheet, linked together using matching Roll Number columns).
  • Main Usage Area: The backbone of modern software systems like online shopping apps, banking databases, and school registry platforms.

✔ Advantages

  • Clean and Simple: Organizing information into basic square tables is highly intuitive.
  • No Data Redundancy: Keeps information completely clean without repeating duplicate data.

✖ Disadvantages

  • Demands High Hardware Power: Running complex multi-table queries requires fast computers and plenty of memory.
