Database Systems Notes

Simple & Short Exam Revision (Class XII)

Centralized Database

A simple database system where all data is stored and kept at a single central location (like one main server machine).

Advantages (Easy to write)

  • Easy Maintenance: Backups and management are easy because everything is in one place.
  • High Security: It is easy to secure because there is only one central entry point.
  • Low Cost: Simple and cheap setup for small organizations.

Disadvantages

  • Single Point of Failure: If the central server crashes, the whole system stops.
  • Traffic Jams: High data traffic slows it down because everyone connects to the same spot.

Distributed Database

A complex database system where data is physically split across different geographical locations and connected by a network.

Advantages (Easy to write)

  • High Availability: If one server crashes, the other locations keep working normally.
  • Load Balancing: User requests are shared across multiple servers, preventing slow-downs.
  • Easy to Expand: Perfect for large companies with many regional branches.

Disadvantages

  • Complex Security: Hard to secure data across multiple networks and locations.
  • Data Sync Issues: Keeping data identical on all servers at the same time is difficult.
  • High Cost: Setting up networks and multiple servers is expensive.

Comparison Table

Basis Centralized Database Distributed Database
Data Location Stored at a single central site. Split across multiple locations.
Main Server Uses only one central server. Uses multiple connected servers.
System Crash If the server fails, the whole system stops. If one node fails, the rest keeps working.
Traffic Rate High traffic congestion at the center. Low traffic congestion (queries are local).
Management Very easy to secure and maintain. Very complex to manage and sync.
Setup Cost Cheaper; best for small scale. Expensive; best for large scale.
