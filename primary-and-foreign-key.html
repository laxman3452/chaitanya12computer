1. Key Features Cheat Sheet

Primary Key (PK) Features
  • Uniqueness: Every value must be completely distinct.
  • No NULLs: Cannot accept missing or blank values.
  • Count: Only one Primary Key allowed per table.
  • Role: Main chosen Candidate Key for row identity.
Secondary / Alternate Key Features
  • Uniqueness: Every value must be distinct.
  • Allows NULLs: Can accept missing or empty values.
  • Count: A table can have multiple secondary keys.
  • Role: Unchosen Candidate Keys left over after picking the PK.

2. Table Demonstration

Notice how the tables link via the Foreign Key, and observe the difference between the Primary Key and Secondary Key in the Students table.

RollNo (PK) AdmissionNo (Secondary Key) StudentName

Result of INNER JOIN
